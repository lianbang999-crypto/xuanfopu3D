# 共修广场 · 全局架构

> 从进站到再联机的完整状态机、各层职责与不变量。
> 本文只记**已实现并有测试覆盖**的行为；未做的列在末节「尚未做」。

---

## 一、一句话架构

**「去哪里」全收进共修大厅；「局中做什么」全收进谱务菜单。**

- **共修大厅**（局外）：一人行谱 / 入座共修室 / 看动态与及第录
- **谱务菜单 ⋯**（局中）：全谱 / 行迹 / 原文 / 玩法 / 设置 / 重开
- **题屏**：冷启动门面，只有「进大厅」或「续局」
- **顶栏题字**：自由观照，职责独立

**退回路径**（每一处都有出口，不把人晾在半路）：

| 从哪里 | 怎么退 | 退到哪 |
|---|---|---|
| 大厅 | 「返回」 | 局中（若在局）／题屏（若无局） |
| 问名卡 / 问密码卡 | 「返回大厅」 | 大厅 |
| 在座局中 | 同修面板「大厅」 | 大厅（**不离席**，自己那室标「您在此」） |
| 在座局中 | 谱务菜单「共修大厅」 | 同上 |
| 在座局中 | 同修面板「离席」 | **自动回大厅**（不停在星图上） |
| 大厅点别室 | 直接点 | **先让出原座**再入新座，不会一人占两室 |
| 及第面板 | 三个去处 | 留座重开／让座回大厅／留座旁观 |

**中断与恢复**（任意状态下）：

| 事件 | 行为 |
|---|---|
| 断线 | 指数退避重连（至多六次），回原座、棋况保留 |
| 断线 > 90 秒 | 座位释放（`OFFLINE_GRACE`） |
| 关页面 | 未送达掷数 `sendBeacon` 送走；存局落 localStorage |
| 再打开 | 题屏主钮变「续掷上局 · 现居某位 · 第 N 掷」，直接回局不经大厅 |
| 房主离席 | 座次最小者递补东位，继承房主之位 |
| 及第后久坐 | 有人求座时被清（`DONE_HOLD` 十分钟） |

## 三、三层职责

| 层 | 文件 | 管什么 | 不管什么 |
|---|---|---|---|
| 广场 DO | `worker/index.js`（固定对象 `__xuanfopu_plaza__`） | 掷轮总数 / 及第录 / 公报流 / 各厅桌位快照 / 锁数上限 | 谱义、规则、行棋判定 |
| 共修室 DO | `worker/index.js`（每室一个，名 = 桌号） | 名单、座次、密码、聊天、转发行棋公报 | 同上 |
| 客户端 | `src/plaza.js` `src/net.js` `src/game.js` | 全部规则判定与谱义呈现 | — |

**不变量**：*规则判定一律在客户端依原谱数据进行；服务端只做名单、密码、转发与留存，不改动任何谱义。*

## 四、编址

| 概念 | 形式 | 说明 |
|---|---|---|
| 厅 | `H{n}` | 一厅 12 室 48 座；满则自动开下一厅，无上限 |
| 室 | `H{厅}T{室}` 如 `H1T12` | 每厅固定 12 室，不动态增减 |
| 座 | 0–3 → 东 / 南 / 西 / 北 | **东位即房主**；东位空出即由座次最小者递补 |
| 珠色 | 金 / 青 / 赭 / 藕 | **跟人不跟座**：递补换座时珠色不变，认得出是谁 |
| 密码 | 四位数字 | 用户自设，加盐哈希（盐 = 桌号）；错满十次封室 |
| 邀请链接 | `#r=H1T5.3333` | 桌号 + 密码，点开即入座，无须手输 |

厅号选择：默认落在**人最多但未坐满**的厅——新人一进来看到的是热闹的厅，不是空厅。

## 五、关键设计决定与理由

**① 无轮次制（自由掷）**
选佛谱同室之间**没有任何相互作用**：我掷的轮不影响你的落位。同座的价值是「看得见彼此的行处」，是共修不是对局。故轮次只是仪式，而它在陌生人场景下要付出「一人挂机全室卡死」的代价——不值得。

**② 重开只清自己（`restart_self`）**
同座各人行处独立，谁也不该被别人一键归零。

**③ 及第不排名次**
本谱纯由掷相所至，比快慢等于奖励运气，且与及第面板自己引的「圓滿菩提。歸無所得。」相悖。同座现况用**陈述式**（「慧安 第19掷行谱中」），不列名次。

**④ 广场公报须用户点了才发**
同室公报自动（在座者本就看得见）；发到全站的及第录**要用户点「录入及第录」**，且按钮下方写明会出现在何处。不经同意不把人的行为推到公共动态。

**⑤ 名字延后到有价值之后才问**
进大厅 / 看动态 / 一人行谱**都不问名**。入座才问（缺省「莲友」），及第时**选填**（不填即「无名同修」）。及第那一刻情绪最高，此时问填写率远高于入口拦一道。

**⑥ 门号不是深浅**
「最深落处」按所依法界的世界 Y 轴算，**不是门号**——门 1「發始因地」是起点门而非最深处。拿门号当深浅会把「上品十惡」误报成比「無間地獄」更深。

**⑦ 桌位快照用推送不用轮询**
参 Colyseus `LobbyRoom`。看广场远比座次变动频繁：轮询要 13 次 DO 请求，推送只要 1 次。

**⑧ 聊天面板不压掷钮**
手机上同修面板是底部抽屉（z=31），掷轮台在 z=10——面板一开就整块盖住掷钮，**掷不了轮**。两处处置：入座后**不自动弹面板**（只出提示，要聊天点左下角脉签）；局中 `body.sfpOn` 让面板抬到掷轮台之上（`bottom: 92px`），全屏态才落到底。

**⑨ 锁数上限**
一厅至多四室设密码（12 之三分一），余下八室永远对陌生人敞开。上锁前向广场查全厅锁况，**查不到就不给锁**——宁可少锁一室，不可锁满全厅。

## 六、防伪与诚信

| 项 | 做法 |
|---|---|
| 掷轮计数 | 只计实际落定的掷；单次上报封顶 60；负数不减总数 |
| 门号 | 越界**剔除**而非夹取——把 99 夹成 15 等于替人捏造「历十五门」 |
| 密码 | 只存加盐哈希；同步消息不带密码；错满十次封室 |
| 桌位快照 | 20 分钟保鲜期，过期即清；行棋时超 5 分钟续报 |
| 座位 | 断线即释放（`liveIds(exceptWs)` 排除正在关闭的连接） |

## 八、尚未做

1. **选佛榜是死代码** — 前端 `openLeaderboard` 及整条 `/api/leaderboard` 链路已无任何入口，职能被大厅的在场人数 + 动态 + 及第录取代。待定：删干净 / 只删前端 / 留着。
2. **冒领座位** — 拿到 `playerId` 即可顶座（boardgame.io 用 `playerCredentials` 解决）。冷启动期风险低。
3. **改密码不踢已在座者** — 已入座者凭 `playerId` 重连不再查密码（有意为之：掉线不该被自己的室挡在外面），故改密码踢不掉已在座的人。
4. **厅选择器 UI** — 后端已支持 `?hall=N`，前端在只有一厅时不显示。等常态坐满 48 人再做。
5. **广场公共聊天** — 未做。治理成本（谤法/拉人/劝募审核）落在运营者身上，等广场热起来再议。
