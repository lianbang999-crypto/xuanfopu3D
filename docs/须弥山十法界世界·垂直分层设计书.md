## 一、技术路线定夺（受权定夺）

**决定：Three.js 连续地形 · 风格化写意世界。不采用方块（voxel）形式。**

理由：

1. **庄严美学**：敦煌矿物色、飞天云气、坛城结构需要连续的形体与光影；方块像素风与「诚敬为底色」的气质相违。
2. **空间语法需要连续地形**：论文立定的「明度一路变亮变暖」「山体右低左高」「光路衔接诸门」，都依赖连续起伏的地形与渐变光照，方块世界无法承载。
3. **工程延续**：现有 `src/explore/scene3d.ts` 已是 Three.js，同栈演进，手机 H5 主通路可行；voxel 引擎（noa 系）基于 Babylon.js，换栈成本高且体积更大。
4. **义理相应**：Minecraft 的核心动词是「挖掘、破坏、建造」——**法界不可由玩家拆改**，此与「不敢妄加增损」的立项铁律相违。我们取「我的世界」之**神**（自由行走、开放世界、发现式探索），不取其**形**（方块与挖掘）。

从「我的世界」继承的三件事：

| 继承项 | 落地方式 |
|--------|---------|
| 自由行走 | 第三人称化身（traveler），触屏摇杆 + 视角拖动 |
| 开放世界 | 分区（chunk）流式加载 + LOD，世界一体无读条 |
| 发现式探索 | 走到才见：远处诸天是剪影与光，走近方显细节与经文 |

### 结构要点

- **六凡在竖轴**（层 1–10：狱鬼畜人天），**净土在横轴**（层 11：西方横超）——「竖出三界难、横超一门易」的净宗判摄，直接成为世界的空间结构。
- **G1 发始因地门 = 南贍部洲上的起手山**。论文通篇设计（起手广场、五段山势、21 道因地山门、右恶左善）**整座平移**进南贍部洲区块，作为玩家入场的第一座山——论文 1.3 所谓「前景入口区」，在世界图上就是南洲。
- **横超光柱**：任何位掷得佛佛可蒙接引（谱之悲心）——表现为**从世界任一角落都能望见的西方金光**，与论文 8.1 第五阶段一致。

## 四、空间语法（承论文五法度，平移至世界尺度）

论文为 G1 立定的法度，在世界尺度上如此展开：

1. **教理为根**：门内小尺度「右恶左善」保持（G1 因地山原样施工）；世界大尺度化为「**下苦上乐、外堕内升、西方横超**」。
2. **明度谱系拉通全界**：论文 7.2 的 L25→L90 由一门拉至全世界——地狱 L25 以下，人间 L55–65，色界 L70+，常寂光最明。玩家的升沉，身体先于文字感知。
3. **圣道在人间起步**：G6 生善滅惡门（忏堂）、G7–G9 三学门（戒坛、禅窟、慧阁）设为**南贍部洲的道场建筑群**——修行本在人间，此是教理，也是动线。
4. **圣道光阶**：G10–G13 位次不作「居住区」而作**出三界的光阶**——自南洲道场升起，于净居天层穿出三界（三果圣人居净居，正合），别教圆教高位没入实报庄严土。声闻缘觉菩萨三法界的「生命形态」以位次光阶呈现，不造世俗化的「圣人聚落」。
5. **数据为契**：探索层只读 `positions.json` / `cosmos.json`，规则层（engine 纯函数）一行不改；世界坐标由新的 `layout-world.ts` 生成，21 单测照跑。
