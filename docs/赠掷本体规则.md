# 《选佛谱》赠掷本体规则

状态：定稿。版本：`grant-ontology-v1.0.0`。生效日期：2026-07-19。

## 唯一生产语义

`贈 N 擲 = 当前操作者获得 N 次即时续掷。`

- 纯赠掷：棋子保持当前位，当前操作者立即续掷 N 次。
- 移位兼赠掷：先移至目的位，再由当前操作者从目的位续掷 N 次。
- 续掷再次获赠时，次数追加到同一操作者的待续掷队列；队列清空后才交轮。
- 单人局与多人局一致，不转赠其他玩家，也不作虚掷。
- 若中途到达确定性终局，关闭剩余待续掷队列。

## 证据边界

《選佛譜》能直接证明某位某轮相是否记作“贈一、贈二、贈三、贈四”，以及该轮是否同时移位；但现存正文没有直接说明多人局受赠者与程序队列。因此“由当前操作者续掷”属于本项目的受控操作解释，权威类型为 `operational_interpretation`，不得写成蕅益大师逐字规定。

智能体回答必须分开：

1. 原谱规则事实：赠几掷、是否移位。
2. 《谱曰》原文解释：只引用可定位原文。
3. 本项目操作规则：同一操作者即时续掷。

禁止再出现“原谱规定赠给其他棋子”“单人局按原谱作虚掷”等说法。

## 数据与验收

当前谱表共有 11 个含赠掷的位、83 个赠掷规则格，其中 79 个纯赠掷、4 个移位兼赠掷。上线前运行：

```bash
npm run check:grant
```

规范机器数据见 `data/grant-ontology-v1.json`。
