# 选佛谱 · 极简设计方案（统一规划）

> 2026-07-20 定稿。约束此后所有 UI/UX、美术改动；改此方案先改本文件。
> 总纲：**星图即棋盘，谱文即内容——界面只做三件事：导航、掷轮、读谱。其余一概退隐。**

## 一 · 设计原则

1. **一屏三区**：左杆（场景导航）· 右杆（十五门天梯）· 底栏（掷轮行动）。中央永远留给星图世界，不放常驻控件。
1a. **单菜单原则**（2026-07-20 定）：全站只有一个菜单＝底栏「⋯」谱务抽屉（拇指区、手机底部抽屉呈现）。右上角☰菜单撤销，其条目并入谱务或裁撤——一切次级功能只此一个入口。
2. **色不过三**：金（#d7aa45 主）· 纸（#efe0b4 文）· 青（#96e1d6 辅，联机/提示专用）。法界配色只在珠色与色点上出现，不做大面积色块。
3. **光即层次**：层级用透明度与发光强弱表达，不用边框粗细；大面积光斑一律 ≤0.42 透明度。
4. **字级五档**：`--fs-xs/sm/md/lg/xl`（11/12.5/14/16/19px）+ 展示级 `--fs-display`（22px）。全站不得出现散点字号；大字模式整体缩放变量。
5. **字族两用**：得意黑（SmileySans）为全站主字；展示级题字（面板标题/落位大字/门介）回退宋体系，气质不塌。
6. **谱义至上**：任何 UI 文案引用谱文必逐字（`scripts/crosscheck-pu.mjs` 护栏）；界面装饰不得遮谱文。

## 二 · 布局架构（一屏三区）

```
┌──────────────────────────────────┐
│ 题字(左上)          （右上无常驻控件）│
│ 场                             天 │
│ 景  ← 中央星图世界（唯一主角） →   梯 │
│ 导                             十 │
│ 航                             五 │
│ (左杆·场景内才现)              门 │
│                                  │
│      [⋯]  长按掷轮  [问]   (底栏) │
└──────────────────────────────────┘
```

- **右杆天梯**（常驻）：十五门刻度+现居门发光；门号一律横排（nowrap）。
- **左杆场景导航**（入场景才现）：色界＝四禅四签；菩萨道场＝十科签。色点+名，无底无框，选中亮点放大。与右杆同语言。
- **底栏**：左「⋯」谱务 · 中「长按掷轮」 · 右「问」。联机时左下加同修脉签。
- **谱务抽屉＝唯一菜单**（原右上☰五项并入后共八项）：
  ① 全谱（十五门二百二十位列表）② 行迹 ③ **观全图／归位**（情境合一：局中在位→观全图拉远；已在外→归位飞回现居）④ 原文（六卷谱文；「参考经典」并为其顶部一签）⑤ 玩法 ⑥ **联机同修**（未入房→大厅；已入房→等候室/聊天；条目副题实时显示房号）⑦ 设置（音效·简繁·特效·大字·低性能·**AI同修**并入此处）⑧ 重开一局（两击确认；联机局中＝全房回等候室）。
  裁撤出菜单：廿一因（因地门介仍可达）· 观照/收藏统计行 · 独立「归位」「参考经典」条目（参考经典＝原文阅读器顶部「所据经论›」）。自由观照底坞补「⋯」——谱务全程一步可达；设置不上题屏，题屏细字行补的是「分享」。
- **探底剖面杆**：只在娑婆全图存在；入门/入场景/入净土一律隐。

## 三 · 15门220位 交互规范

- **掷轮**：长按默念→松手旁掷（占察轮 3D 实转）；未轮到（联机）按钮压暗不可掷。
- **判词窗**（每掷必出，中央下方）：`轮面二字 → 方向徽 → 去向位名（展示级字）→ 谱曰缘由（可展开原文）→ 行▸`。点正文展白话+原文；下滑收签、上滑唤回。
- **行棋**：彗星光弧乘行，跨门先拉远同框再进；落位金环涟漪+位名大字（展示级）。
- **位次浏览**：点门星展本门位珠；位珠点开＝谱注抽屉（原文+白话+出处行号）。
- **不打断原则**：一切浮层可点外关闭；行棋动画可点跳过（设置「行棋特效」关＝自动直达）。

## 四 · 十法界世界美术规范（极简）

- **须弥山**：敦煌细腰旋成体，四宝四面（南琉璃/东白银/北黄金/西颇胝迦），金线描边为唯一描边语言。
- **七金山七香水海**：金脊环（皆金所成，双金调相间）×7，环距内山基不叠压，海环必可见。
- **四洲**：东半月/南车箱/西满月/北方座+各二中洲；洲色映本面宝色，昼夜随日。
- **色界四禅**：光盘+星环，盘晕 ≤0.42、侧晕减半；成员星只在该层绽开时亮名。
- **无色界**：无方所——只以轴顶微光与名示意，不做实体建筑。
- **净土**：横轴西方（横超光路），敦煌壁画底幕+莲池双层金碧微光。
- **降噪三禁**：禁常亮大光斑、禁多层重叠辉光、禁与谱文争眼的动画。

## 五 · 卡片（overlay）统一体系规范（2026-07-21 重定，v225–v236）

> 用户三条定案连成一套统一卡片语言：①**白话为主体，原文为依据**（不再以繁体原文当主句）；②**一卡一序、体例同构**（全站卡片共用一套排版类与结构次序）；③**每张卡都能回到局中**（局中态主钮＝「回到局中」，非局中＝「关闭」）。

### 5.0 命名（2026-07-21 用户定案·大白话原则）
- 「谱曰原文」一律改叫 **「原文」**；「对读」一律改叫 **「文白对照」**。原则：用大白话让用户一眼明白是什么，术语让位于清楚。

### 5.1 统一壳与出场
- 壳：`.panel`（圆角14 / 边线金 26% / 磨砂）＋ 遮罩；手机端谱务/联机类走**底部抽屉**（`.ovsheet` + 抓手）。
- 一屏一窗：新窗必关旧窗；✕ 与蒙层点击等效；场景转场用 `softCloseOverlay`（不吞 `.keepOv` 重要面板）。

### 5.0b 内容极简·信息只出一次（2026-07-21 用户定案）
- **不重复信息**：同一信息、标签、内容全站只显示一次。门名只在**顶签**出现（位卡「解读」不再重述「所在门：第X门『门名』」）；位名只在标题；「所在法界」仅当法界名 ≠ 位名时才显（免 无想天/无想天 自我重复）。
- 位卡去冗：撤「AI 解读」折叠壳与「本门总说」折叠（门总说原文经顶签「本門譜文 ›」入卷阅读，不在每张位卡重复）；门大意收成一行**键值**（`.cKv`：小金键「本门大意」+ 白话值），不再叙述式重述。
- 「问」聊天卡撤底部说明脚注（用户点名删）。

### 5.1a 双主题（2026-07-21 用户定案）：暗夜（默认）/ 写经纸（可切）
- **暗夜用底色渐变作分层**（用户点单）：`--ck-panel` 改 `linear-gradient(168deg, 顶 rgba(40,33,64) → 底 rgba(16,13,30))`，卡片有纵深，段落坐在渐变上而非一片死黑；写经纸同理用暖渐变（纸有厚度）。层次＝渐变底 + 原文沉入凹槽块 + 文字三级明度，三管齐下。
- **问题**：旧暗底把白话/原文/对照/注脚都做成暗底上的中间调，糊在一起、不好读。**两手治理**：
- **① 暗夜主题（默认·层次重做）**：白话提亮到近白（`--ck-plain` #f6ecc6）、解读降一档、注脚最弱，三级明度拉开；原文进**深凹槽块**（`.verse` bg `rgba(0,0,0,.26)`+金左线）与白话分开；白话对照石绿提亮（`--ck-bai` #93cbac）并标「白」。
- **② 写经纸主题（可切·最好读）**：暖浅底墨字——卡底 #f7efdc，正文墨 #2a2318，原文沉香墨 #4a3d24 沉入凹槽块，白话对照石绿 #2f6a52。卡片浮于暗世界之上如展卷。
- **实现**：全套卡片色走 `--ck-*` CSS 变量；`html.paperCards` 整体切换（`save.cardTheme='night'|'paper'`，设置卡「卡片主题」开关，`applyCardTheme()` 启动应用）。**只切 token 不改结构**。世界标签/控制台/掷轮钮不在卡内，不随主题变；写经纸模式对旧卡片内联浅色有 `[style*=]!important` 兜底映射，保证纸上可读。

### 5.2 排版体例（六个类，全站统一；色走 `--ck-*` 双主题 token，字级归五档变量）
| 类 | 用途 | 暗夜 / 写经纸 |
|----|------|------|
| `.cMeta` | 金字顶签（门次/位次/卷次/分节标题） | 亮金 / 泥金 |
| `.cPlain` | 白话主体首行（「这是什么」一句） | 近白 / 墨 |
| `.cRead` | 解读层（在何门·何界·为什么） | 米黄 / 沉墨 |
| `.verse` | **原文**（沉入凹槽块，宋体） | 沉香黄+暗块 / 沉香墨+浅块 |
| `.dd` | **文白对照**白话对句（标「白」，缀原文段下） | 石绿亮 / 石绿深 |
| `.cNote` | 灰字注脚（出处体例/操作提示） | 灰金 |
| `.lnk` | 点读虚线（可点入原文/法界/卷文） | 随主题 |

### 5.3 位卡三层一序（`openSfpNote`，v228 定 / v237 改名）
自上而下恒定：**顶签 `.cMeta`（门·第N/220位·卷次点入）→ 白话首行 `.cPlain` → 「解读」折叠（所在门＋所在法界＋是否毕局，`.cRead`）→ 「升降行法·廿一组轮相」折叠 → 「原文 · 文白对照」折叠（`duiduHtml`：从 `SFP_CANON_DOORS` 读取该位完整谱注，白话库键逐字嵌入原文；命中段「原文在上 `.verse`、白话对句在下 `.dd`」，出处 `.cSrc`）→ 「本门总说」折叠 → 上/下位翻页 → 主钮**。
- 原则：**先白话后原文、先大意后逐字、先解读后依据**；引文皆逐字繁体（CBETA B0136），文白分层不混、两种颜色两种声音。

### 5.4 判词卡（`showVerdict`，v226 定）
- 白话为主句（`#vWhy`）；命中原文库时缀「谱曰原文 ▸」（`.vsrcT`）点开小字原文（`#vSrc`），不再有独立工具行。
- 去处位名（`.vdst`）**可点弹白话小签**（`openPosGloss`：位名＋一句白话＋「原文说明 ›」入位卡）。
- 「AI 解读」小签（`.vaskC`）缀在正文同行，点开＝直达本掷的既定解读内容卡（本地检证拼证，非聊天）；卡内可另点「问」进入追问。

### 5.4b 「问」参考 wenchao 问文钞（2026-07-21 用户点单，NotebookLM＝谷歌 NotebookLM）
- 参照 `lianbang999/wenchao` 的 `/ask` 与 `ai-core.js`：**据文检证 + 可点出处**的问答范式（谷歌 NotebookLM 式——答语只据所供语料、句句挂出处、点角标看原文摘录）。
- 已采：**智能体三段式检索反馈**（`.cbStage`：「正在检证 G 版经据库」→ 过 5 秒「正在细检 G 版相关篇章并综合」，仿文钞 slowTimer，免十余秒静句被当卡死）；输入框跟随主题（`.cbInput`，16px 防缩放）；撤底部说明脚注。
- 每次普通提问直接接现有智能体（`/api/ask`），答复带出处、明标 AI 生成、每人每日 100 次；本地检证同步生成，作为等待期间速查及连接失败兜底。

### 5.5 「问」＝与本谱对话（聊天式，v236 定）
- 聊天双气泡（问右 `.cbU` / 答左 `.cbA`）＋ 快问签（`.chipQ`）＋ 输入行（16px 防 iOS 缩放）。
- 与「AI 解读」严格分家：打开「问」不自动灌入上一掷解读；默认 3 个快问随局势生成——当前谱位、最近轮相、所在门各一问；未开局时显示玩法、六字轮相、上品上生三问。
- **G 版智能体主答**（2026-07-22）：普通聊天每一问都直接交自建 Cloudflare 智能体（`/api/ask` → `ASK_SERVICE` → `xuanfopu-evidence-agent`），其经据索引来自 `选佛谱G版（codex）/agent/corpus/rag-v2`；答复以气泡回贴，**明标「AI 生成」与引文出处**，本机缓存同问即答。结构化本地检证（逐字谱曰·行法表·813 句缘由白话对读库·220 位白话·词典）保留为等待/离线兜底，并折叠在智能体答复下供核对。
- 聊天输入框占位提示与快捷问题签统一使用全局字体变量 `--f-ui`，不再由浏览器默认输入字体造成视觉跳变。

### 5.6 卡片「回到局中」通则（v221/v236）
局中（`sfpS.active`）时，行迹/消息/全图/原文/净土/廿一因/选佛榜/谱务各卡主钮一律「回到局中」；非局中为「关闭」。谱务菜单亦补「回到局中」大钮。

## 七 · 联机方案（极简定案）

**取舍**：不做公共大厅房间列表——本作是莲友私局共修，陌生人房列表带来审核与骚扰负担，与气质不合。
「大厅」＝一屏三事：**开新房 / 房号入房 / 最近的房**；传播靠**邀请链接**，不靠公开曝光。

联机三幕：**大厅 →（入房）等候室 →（开局）脉签+聊天**。

- **大厅**（题屏「联机同修」进入）：名号（一次填写永久记住）→ 大按钮「开新房」；或房号+入房；下方「最近的房」至多三枚（实时显示 在线/在房 人数、已开局），点即再入。
- **等候室**（入房未开局的主场景）：房号大字（点击复制）· 四座横排可视化（实座＝珠色圆珠+名号，房主带「主」标，离线淡去；空座＝虚线圈「虚位」）· **情境主按钮**（独自→「邀请莲友·点开即入」；房主且≥2人→「开局·N位同修」；非房主→「候房主开局·可先邀莲友」；四座已齐非房主→置灰候开局）· 细字行「聊天·离房·收起」。开局即自动散场；点蒙层/收起可暂离（脉签点回）。
- **同修脉签**（原「聊」浮钮升级）：左下小胶囊内**四座色点即名单**——谁在房一目了然；开局后**轮到谁谁的点脉动发光**；离线点淡去；未读角标照旧。点开＝未开局回等候室、开局后开聊天。
- **房号＝4 位数字**（2026-07-20 定，为微信/社交传播设计）：口头可报、语音可念、手机可拨。旧字母房号与旧深链仍兼容（正则收 `[A-Za-z0-9]{4,8}`）。
- **邀请＝分享卡**（一个组件两用）：带房号＝邀请卡（房号大字＋**二维码**＋「转发给莲友」主钮）；无房号＝荐游戏卡（题屏细字行「分享」）。二维码内嵌生成（无外链库），面对面场景莲友扫屏即入，零输入。转发走 `navigator.share`（系统分享面板）；**微信内置浏览器无 share**——检测 `MicroMessenger` UA 时改出「点右上角 ⋯ 转发给朋友」引导＋复制文案兜底。等候室常显小二维码。
- **一键入局**：深链 `#r=1234` 落地——本机存有名号（`sm10.net.name`）即**全自动入房入局**（toast 报房号）；首次仅弹一张单栏名号卡（「莲友邀您入房 1234」＋名号＋大按钮「入局」）。入房即自动入局，聊天/等候室随之。
- **重开一局＝回等候室**（联机中）：任一同修（房主）点「重开一局」→ 服务端 `restart` 广播 → 全房清棋况、`started=false`、回等候室——座次保留、可再邀新莲友、房主再点开局。独修重开照旧两击确认。及第面板同理提供「同房再来一局」。
- **聊天**：名单与聊天同面板；**快语行**「南無阿彌陀佛」「隨喜讚歎 🙏」一键即发（共修常用语，手机免打字）。**手机端＝底部半屏抽屉**（宽 100%、约 60vh、圆角顶＋抓手，上滑拉全屏、下滑收起）；输入框字号 ≥16px（防 iOS 聚焦自动放大）；键盘弹起以 visualViewport 贴合。桌面端浮窗照旧。
- **轮次**：开局后按座次轮掷；未轮到者掷轮压暗、脉签与名单亮当前者；贈掷续掷不交轮（依赠掷本体规则定稿）。
- **断线**：自动重连回原座、棋况保留；离线者名单置灰不除名。

## 七之二 · 功耗规范（v221 发热治理，2026-07-20 移植）

- 触屏机（`pointer:coarse`）默认省电档：DPR 上限 1.6（桌面 2）；泛光缓冲减半分辨率。
- 静观期 30fps：飞行/转场/掷轮/触控输入/远端珠滑行等动势期放行全帧率；交互后 1 秒提帧窗。
- 自适应降档：30fps 仍撑不住（帧距 >45ms 持续约 7 秒）则 DPR 再降 15%，只降不升防振荡（下限 0.7）。
- 探针：`__dbg.perf` 读 `{isCoarse, dprScale, pr}`。新增动画必须自问「静观期是否在烧帧」。
